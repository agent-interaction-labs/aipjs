UI Mirroring

When an agent interacts with your site, mirroring shows you exactly what's happening in real time. Every tool call, every field interaction, every value change — all surfaced as visual feedback for website owners. You see what the agent sees.

Visual Indicators

  • Blue dot (bottom-right corner) — pulses green when the agent is actively working, static blue when idle. Always present as a subtle status indicator.
  • Element highlighting — affected fields receive a blue outline for 1.5 seconds, making it obvious which element the agent is interacting with.
  • Form population — agent-provided values appear directly in the actual form fields. You can watch forms fill out in real time as the agent works.

How It Works

Mirroring is event-driven. Three core hooks manage the visual feedback loop:

  • onToolStart() — adds the focus CSS class to the target element and mirrors the initial value. This is when the blue outline appears.
  • onToolComplete() — removes all visual indicators after the tool finishes. The outline fades and the dot returns to idle.
  • watchNavigation() — re-scans the DOM on SPA route changes so mirroring stays accurate across page transitions.

mirrorElementValue()

Manual API for mirroring any element's value. Useful when you need precise control over what the user sees during agent interaction.

Signature: mirrorElementValue(selector, value)

Supports all standard form elements:

  • <input> — text, number, email, password, etc.
  • <select> — sets the selected option
  • <checkbox> — toggles checked state
// Mirror a value into a text input
aip.mirrorElementValue('#search-box', 'wireless headphones');

// Mirror a select dropdown
aip.mirrorElementValue('#category-select', 'electronics');

// Mirror a checkbox
aip.mirrorElementValue('#agree-terms', true);

CSS Classes

All mirroring classes use the configured cssPrefix (default: "aipjs"):

Class Effect
.aipjs-mirror-focus Blue outline on the targeted element, visible for 1.5 seconds
.aipjs-mirror-changed Brief flash animation when a value is programmatically set
#aipjs-indicator The bottom-right dot indicator (pulses green when active)

SPA Support

Single-page applications receive special treatment. Mirroring stays accurate even through client-side navigation:

  • watchNavigation() intercepts pushState and replaceState calls, and listens for popstate events.
  • Automatic re-scan: calls refreshTools() on every route change so the agent always sees the current page state.
  • MutationObserver: watches for DOM changes caused by framework re-renders (React, Vue, Svelte), ensuring indicators stay attached to the right elements.

Disabling

To turn off all visual mirroring, set uiMirroring: false in your AIP configuration. The agent will still function normally — you just won't see the visual feedback. 

const aip = new AIP({
  uiMirroring: false
});
