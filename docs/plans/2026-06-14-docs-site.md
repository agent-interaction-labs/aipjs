# Docs Site Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Multi-page documentation site for aip.js with sidebar navigation, comprehensive content covering all SDK features, and demo integration. Served via GitHub Pages from `/docs/`.

**Architecture:** All pages share a single shell (nav + layout) via a shared CSS file and a lightweight JS module that injects the navigation sidebar. Each page is a standalone HTML file with its own content section. No build step — vanilla HTML/CSS/JS. Dark theme matching the demo design system.

**Tech Stack:** Vanilla HTML5, CSS3 (shared design tokens), ES modules for nav injection. Pages import nothing except the shared nav script and CSS.

**Live at:** `https://aipjs.github.io/aip.js/docs/` with demo at `https://aipjs.github.io/aip.js/demo/`

**Step 3: Create `docs/assets/nav.js`**

Lightweight module that injects the sidebar into any docs page:

```js
// docs/assets/nav.js — injects shared sidebar navigation
(function() {
  const nav = document.getElementById('docs-nav');
  if (!nav) return;

  nav.innerHTML = `
    <div class="nav-section">
      <div class="nav-section-title">Getting Started</div>
      <a href="/aip.js/docs/" class="nav-link">Introduction</a>
      <a href="/aip.js/docs/getting-started.html" class="nav-link">Installation</a>
    </div>
    <div class="nav-section">
      <div class="nav-section-title">Core Concepts</div>
      <a href="/aip.js/docs/auto-inference.html" class="nav-link">Auto-Inference</a>
      <a href="/aip.js/docs/manual-config.html" class="nav-link">Manual Config</a>
      <a href="/aip.js/docs/hitl-security.html" class="nav-link">HITL Security</a>
      <a href="/aip.js/docs/ui-mirroring.html" class="nav-link">UI Mirroring</a>
    </div>
    <div class="nav-section">
      <div class="nav-section-title">Plugins</div>
      <a href="/aip.js/docs/ecommerce-plugin.html" class="nav-link">E-Commerce</a>
      <a href="/aip.js/docs/custom-plugins.html" class="nav-link">Custom Plugins</a>
    </div>
    <div class="nav-section">
      <div class="nav-section-title">Reference</div>
      <a href="/aip.js/docs/configuration.html" class="nav-link">Configuration</a>
      <a href="/aip.js/docs/webmcp.html" class="nav-link">WebMCP Compat</a>
      <a href="/aip.js/docs/api-reference.html" class="nav-link">API Reference</a>
    </div>
    <div class="nav-section">
      <a href="/aip.js/demo/" class="nav-link demo-link">→ Live Demo</a>
      <a href="https://github.com/aipjs/aip.js" class="nav-link demo-link">→ GitHub</a>
    </div>
  `;

  // Highlight current page
  const path = window.location.pathname;
  nav.querySelectorAll('.nav-link').forEach(link => {
    if (link.getAttribute('href') === path) link.classList.add('active');
  });
})();
```

## Task 5: HITL Security (`docs/hitl-security.html`)

**Objective:** The security model. Risk classification, HITL modal behavior, why it's non-bypassable.

**Content outline:**
- Security model overview: SAFE vs HIGH_RISK classification
- How risk is determined (auto-inference heuristics + explicit `data-aip-*` attributes)
- HITL modal lifecycle:
  1. Agent invokes HIGH_RISK tool
  2. Page freezes
  3. Modal appears (z-index 2147483647)
  4. Human approves or denies
  5. Result returned to agent (or blocked)
- Configuration: `hitlTimeout`, `hitlRiskLevels`
- Prompt injection sanitization — what it protects against
- Custom risk policies (permissive, strict, lockdown)
- Audit trail
